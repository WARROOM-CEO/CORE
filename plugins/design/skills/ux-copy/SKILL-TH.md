# UX Copy Writing

**ประเภท**: User-invoked
**Trigger**: `/ux-copy` + context or copy to review

## ภาพรวม

เขียนหรือ review UX copy ในบริบท interface ใด ๆ: microcopy, error messages, empty states, CTAs, confirmation dialogs, tooltips, loading states, onboarding

## Principles

1. **Clear**: พูดสิ่งที่คุณหมายถึงอย่างแน่นอน ไม่มี jargon
2. **Concise**: ใช้คำเท่าที่ต้อง
3. **Consistent**: ใช้ terms เดียวกันสำหรับสิ่งเดียวกัน
4. **Useful**: คำทุกคำช่วยผู้ใช้บรรลุเป้าหมาย
5. **Human**: เขียนเหมือนคนจริง ไม่ใช่โรบอต

## Copy Patterns

**CTAs**: Start with verb ("Save changes", "Create account") ซ้ำขอบเขต outcome
**Error messages**: What + Why + How to fix ("Payment declined. Your card was declined by your bank. Try a different card.")
**Empty states**: What + Why empty + How to start
**Confirmation**: Clear action ("Delete 3 files?") describe consequences label buttons with action
**Tooltips**: Concise, helpful, never obvious
**Loading states**: Set expectations, reduce anxiety
**Onboarding**: Progressive disclosure

## Output

Copy recommendation ที่มี:
- Recommended copy
- Alternatives table (copy, tone, best for)
- Rationale (user context, clarity, action-orientation)
- Localization notes
