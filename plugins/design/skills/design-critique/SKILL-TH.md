# ประเมินผล Design

**ประเภท**: User-invoked
**Trigger**: `/design-critique` + Figma URL, screenshot, หรือ description

## ภาพรวม

ให้ structured feedback ด้านการออกแบบในหลายมิติ ติดตามกรอบการประเมินผล:
- **First Impression**: ว่าดึงสายตาไปยังที่ใด ความเข้าใจวัตถุประสงค์
- **Usability**: ผู้ใช้สามารถบรรลุเป้าหมายได้หรือไม่ ขั้นตอนไม่จำเป็น
- **Visual Hierarchy**: ลำดับการอ่าน การใช้ whitespace ประสิทธิภาพของ typography
- **Consistency**: ปฏิบัติตามระบบ design หรือไม่ ส่วนประกอบที่คล้ายกันทำงานเหมือนกันหรือไม่
- **Accessibility**: color contrast, touch targets, readability

## Output

Critique report ที่มี:
- Overall impression (1-2 ประโยค)
- ตารางผลการค้นหา usability พร้อม severity
- Visual hierarchy insights
- ตารางช่องว่าง consistency
- การตรวจสอบ accessibility
- สิ่งที่ทำได้ดี
- ลำดับความสำคัญในการแนะนำ

## เชื่อมโยงกับ

- **accessibility-review**: ตรวจสอบ accessibility อย่างลึกซึ้ง
- **design-handoff**: เตรียม design สำหรับ handoff
- Figma MCP: ดึง design โดยตรงและเปรียบเทียบกับระบบ design
- User feedback MCP: ข้างต้องอ้างถึง user feedback และปัญหา support
