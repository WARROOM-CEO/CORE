# Example Skill — ฉบับภาษาไทย

เอกสารนี้เป็นคำแปลภาษาไทยสำหรับผู้ใช้ที่ต้องการเข้าใจโครงสร้างและแนวทางการสร้าง Skill สำหรับ Claude Code Plugin

---

## ภาพรวม

Skill คือความสามารถที่ Claude เรียกใช้เองโดยอัตโนมัติตามบริบทของงาน ต่างจาก Command (ที่ผู้ใช้เรียกเอง) หรือ Agent (ที่ Claude สร้างขึ้น) Skill ให้คำแนะนำเชิงบริบทที่ Claude นำไปรวมกับคำตอบ

---

## เงื่อนไขการเรียกใช้ Skill นี้

Skill นี้จะทำงานเมื่อผู้ใช้ต้องการ:
- สร้างหรือทำความเข้าใจ Plugin Skill
- ต้องการเทมเพลตหรือตัวอย่าง Skill
- พัฒนา Pattern สำหรับ Skill

---

## โครงสร้างไฟล์

### ไฟล์ที่จำเป็น

```
skills/
└── ชื่อ-skill/
    └── SKILL.md          # ไฟล์หลักของ Skill (จำเป็นต้องมี)
```

### ไฟล์เสริม (ไม่บังคับ)

```
skills/
└── ชื่อ-skill/
    ├── SKILL.md          # ไฟล์หลัก (ภาษาอังกฤษ + ไทยสำหรับส่วนที่สื่อกับผู้ใช้)
    ├── SKILL-TH.md       # คำแปลภาษาไทยสำหรับผู้ใช้อ่าน
    ├── README.md         # เอกสารเพิ่มเติม
    ├── references/       # เอกสารอ้างอิง
    │   └── patterns.md
    ├── examples/         # ตัวอย่างไฟล์
    │   └── sample.md
    └── scripts/          # Script ช่วยเหลือ
        └── helper.sh
```

## Frontmatter Options

Skill รองรับ field เหล่านี้ใน frontmatter:

| Field | จำเป็น | รายละเอียด |
|-------|--------|-----------|
| `name` | ✅ | ตัวระบุ Skill — แนะนำให้เพิ่ม suffix `-th` เพื่อแยกจาก Official skill |
| `description` | ✅ | เงื่อนไขการเรียกใช้ — ใส่ทั้งภาษาไทยและอังกฤษเพื่อให้ครอบคลุม |
| `version` | ❌ | หมายเลขเวอร์ชัน เช่น `1.0.0` |
| `license` | ❌ | ข้อมูลสัญญาอนุญาต |

---

## การเขียน Description ที่ดี

Field `description` มีความสำคัญมาก — มันบอก Claude ว่าควรเรียก Skill นี้เมื่อไหร่

**รูปแบบที่แนะนำ:**
```yaml
description: This skill should be used when the user asks to "วลีเฉพาะ", "another phrase", mentions "คำสำคัญ", or discusses หัวข้อ.
```

**ควรรวม:**
- วลีที่ผู้ใช้มักพูด (ทั้งไทยและอังกฤษ)
- คำสำคัญที่บ่งชี้ความเกี่ยวข้อง
- หัวข้อที่ Skill ครอบคลุม

---

## แนวทางการเขียนเนื้อหา Skill

1. **ระบุวัตถุประสงค์ชัดเจน** — บอกว่า Skill ช่วยเรื่องอะไร
2. **กำหนดเงื่อนไขการใช้** — ระบุว่าควรเรียกใช้เมื่อไหร่
3. **จัดโครงสร้างให้เป็นระเบียบ** — จัดข้อมูลอย่างมีตรรกะ
4. **คำแนะนำที่ปฏิบัติได้จริง** — ให้ขั้นตอนที่ชัดเจน
5. **ตัวอย่างประกอบ** — รวมตัวอย่างจริงเมื่อเหมาะสม

---

## แนวปฏิบัติที่ดี

- ให้ Skill เน้นโดเมนเดียว อย่าพยายามทำหลายอย่างในที่เดียว
- เขียน description ที่บ่งชี้ชัดว่าเมื่อไหร่ควร activate — ใส่วลีภาษาไทยด้วย
- เก็บเอกสารอ้างอิงไว้ใน subdirectory สำหรับ Skill ที่ซับซ้อน
- เพิ่ม suffix `-th` ต่อท้ายชื่อ Skill เพื่อหลีกเลี่ยงการชนกับ Official skill
- ทดสอบว่า Skill ทำงานกับ query ที่คาดไว้ทั้งภาษาไทยและอังกฤษ
- หลีกเลี่ยงการซ้อนทับกับเงื่อนไขการเรียกใช้ของ Skill อื่น
