# cowork-plugin-management-th

**ปลั๊กอิน Cowork สำหรับสร้างและปรับแต่งปลั๊กอิน — เวอร์ชันภาษาไทย**

ปลั๊กอินนี้ช่วยให้คุณสร้างปลั๊กอินใหม่ตั้งแต่ต้น และปรับแต่งปลั๊กอินที่มีอยู่ให้เข้ากับเครื่องมือและเวิร์กโฟลว์ขององค์กร โดย Claude จะสื่อสารกับผู้ใช้เป็น**ภาษาไทย**ตลอดการใช้งาน

---

## Skills ที่มีในปลั๊กอินนี้

### `/cowork-plugin-customizer-th` — ปรับแต่งปลั๊กอิน

ใช้เมื่อต้องการปรับแต่งปลั๊กอินที่มีอยู่แล้วให้เหมาะกับองค์กรของคุณ

**เรียกใช้ด้วย:**
- "ปรับแต่งปลั๊กอิน"
- "ตั้งค่าปลั๊กอิน"
- "กำหนดค่าปลั๊กอิน"
- "customize plugin" / "set up plugin"

**สิ่งที่ทำได้:**
- ตั้งค่าปลั๊กอินเทมเพลต (แทนที่ค่า placeholder ด้วยข้อมูลจริงขององค์กร)
- ปรับแต่งเฉพาะส่วน เช่น connector, skill หรือเวิร์กโฟลว์
- ค้นหาและเชื่อมต่อ MCP เซิร์ฟเวอร์ที่เหมาะกับเครื่องมือที่ใช้
- สรุปผลการปรับแต่งเป็นภาษาไทย

---

### `/create-cowork-plugin-th` — สร้างปลั๊กอินใหม่

ใช้เมื่อต้องการสร้างปลั๊กอินใหม่ตั้งแต่ต้นผ่านการสนทนา

**เรียกใช้ด้วย:**
- "สร้างปลั๊กอิน"
- "ทำปลั๊กอินใหม่"
- "พัฒนาปลั๊กอิน"
- "create a plugin" / "build a plugin"

**ขั้นตอนที่ Claude จะพาไป:**
1. **การค้นพบ** — ทำความเข้าใจสิ่งที่ต้องการสร้าง
2. **การวางแผนส่วนประกอบ** — กำหนดว่าต้องการ skill, agent, hook หรือ MCP ใดบ้าง
3. **การออกแบบ** — ระบุรายละเอียดแต่ละส่วนประกอบ
4. **การสร้าง** — สร้างไฟล์ปลั๊กอินทั้งหมด
5. **การส่งมอบ** — แพ็กเกจเป็นไฟล์ `.plugin` พร้อมติดตั้ง

---

## โครงสร้างไฟล์ (Bilingual Design)

ปลั๊กอินนี้ออกแบบให้ประหยัด token สูงสุด โดยแยกไฟล์ตามผู้ใช้งาน:

| ไฟล์ | ภาษา | ใช้โดย | วัตถุประสงค์ |
|------|------|--------|--------------|
| `SKILL.md` | อังกฤษ + ไทย (ส่วนสื่อกับผู้ใช้) | Claude | ลดการใช้ token ขณะรัน |
| `SKILL-TH.md` | ไทยทั้งหมด | ผู้ใช้ | เปิดอ่านทำความเข้าใจ |
| `references/*.md` | อังกฤษ | Claude | อ้างอิงขณะรัน |
| `references/*-th.md` | ไทย | ผู้ใช้ | เปิดอ่านประกอบ |

---

## การติดตั้ง

1. ดับเบิลคลิกไฟล์ `cowork-plugin-management-th.plugin` ใน Cowork
2. กด **ยืนยันการติดตั้ง** ในหน้าต่าง preview
3. เรียกใช้ได้ทันทีด้วยคำสั่งข้างต้น

> **หมายเหตุ:** หากติดตั้ง Official plugin (`cowork-plugin-management`) ไว้ด้วย ทั้งสองจะทำงานแยกกันโดยไม่ขัดแย้ง เนื่องจากชื่อ plugin และ slash command ต่างกันทั้งหมด
