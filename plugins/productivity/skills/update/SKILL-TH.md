# สกิล: อัปเดต (`/productivity:update`)

**ซิงค์งานและรีเฟรชหน่วยความจำจากกิจกรรมล่าสุด**

---

## ใช้เมื่อไหร่?

- ดึงงานที่ได้รับมอบหมายใหม่จากระบบติดตามงานมาใส่ใน TASKS.md
- คัดแยกงานที่ค้างอยู่นานออก
- เติมช่องว่างในหน่วยความจำ (บุคคลหรือโครงการที่ยังไม่รู้จัก)
- สแกนเชิงลึกเพื่อจับรายการสิ่งที่ต้องทำที่ฝังอยู่ในระบบแชทและอีเมล

---

## สองโหมด

```
/productivity:update                  ← โหมดปกติ
/productivity:update --comprehensive  ← สแกนเชิงลึกทุกแหล่ง
```

---

## โหมดปกติ (ค่าเริ่มต้น)

### 1. โหลดสถานะปัจจุบัน
อ่าน `TASKS.md` และโฟลเดอร์ `memory/` ถ้าไม่มี → แนะนำให้รัน `/productivity:start` ก่อน

### 2. ซิงค์งานจากแหล่งภายนอก
ตรวจสอบระบบติดตามงานที่เชื่อมต่อ (เช่น Asana, Linear, Jira) และดึงงานที่ได้รับมอบหมายมาเปรียบเทียบกับ TASKS.md:

| สถานะ | การกระทำ |
|-------|---------|
| มีในแหล่งภายนอก ไม่มีใน TASKS.md | เสนอเพิ่ม |
| มีใน TASKS.md ไม่มีในแหล่งภายนอก | แจ้งว่าอาจเป็นงานที่ค้างเก่า |
| เสร็จสมบูรณ์ในแหล่งภายนอกแต่ยังอยู่ในกำลังดำเนินการ | เสนอทำเครื่องหมายว่าเสร็จ |

### 3. คัดแยกงานที่ค้างเก่า
ตรวจสอบงานในรายการกำลังดำเนินการและแจ้งเตือน:
- งานที่เลยกำหนดเวลาแล้ว
- งานที่อยู่ในรายการกำลังดำเนินการมากกว่า 30 วัน
- งานที่ไม่มีบริบท

เสนอทางเลือก: ทำเครื่องหมายว่าเสร็จ / เลื่อนกำหนดเวลา / ย้ายไปงานในอนาคต

### 4. เติมช่องว่างในหน่วยความจำ
สำหรับแต่ละงาน Claude จะถอดรหัสทุกรายการ (บุคคล โครงการ คำย่อ) และรายงานสิ่งที่ยังไม่รู้จัก:

```
พบคำที่ยังไม่มีข้อมูล:

1. "Phoenix" (จาก: "ส่งรายงานสรุปสถานะโครงการให้ Todd เรื่องอุปสรรค Phoenix")
   → Phoenix คืออะไร?

2. "Maya" (จาก: "ประสานงานกับ Maya เรื่องการออกแบบ API")
   → Maya คือใคร?
```

### 5. รายงานผล
```
อัปเดตเสร็จสมบูรณ์แล้ว:
- งาน: +3 จาก Asana, 1 รายการทำเครื่องหมายว่าเสร็จ, 2 รายการคัดแยกออก
- หน่วยความจำ: เติมข้อมูล 2 รายการ, อัปเดต 1 โครงการ
- งานทั้งหมดถอดรหัสได้แล้ว ✓
```

## โหมดเชิงลึก (`--comprehensive`)

ทุกอย่างในโหมดปกติ บวกกับ:

### สแกนกิจกรรมจากทุกแหล่ง
- **ระบบแชท** — ข้อความล่าสุด ช่องทางที่ใช้งานอยู่
- **อีเมล** — ข้อความที่ส่ง
- **เอกสาร** — เอกสารที่เปิดล่าสุด
- **ปฏิทิน** — การประชุมล่าสุดและที่จะมาถึง

### แจ้งรายการสิ่งที่ต้องทำที่อาจพลาด
```
งานที่อาจยังไม่ได้บันทึก:

1. จากแชท (18 ม.ค.):
   "ฉันจะส่ง mockup ที่อัปเดตแล้วภายในศุกร์"
   → เพิ่มใน TASKS.md?

2. จากการประชุม "การประชุมประจำวัน Phoenix" (17 ม.ค.):
   มีการประชุมประจำแต่ไม่มีงาน Phoenix ในรายการกำลังดำเนินการ
   → มีงานที่ต้องทำอยู่ไหม?
```

### เสนอหน่วยความจำใหม่
```
บุคคลใหม่ที่ยังไม่มีในหน่วยความจำ:
| ชื่อ | พบกี่ครั้ง | บริบท |
|------|-----------|-------|
| Maya Rodriguez | 12 | การออกแบบ การตรวจสอบ UI |

โครงการ/หัวข้อใหม่:
| ชื่อ | พบกี่ครั้ง | บริบท |
|------|-----------|-------|
| Starlight | 15 | เอกสารวางแผน ผลิตภัณฑ์ |

แนะนำให้ล้างข้อมูล:
- โครงการ Horizon — ไม่มีการพูดถึง 30 วันแล้ว เสร็จสมบูรณ์แล้วหรือยัง?
```

## หมายเหตุสำคัญ

- **ไม่เพิ่มงานหรือหน่วยความจำโดยไม่ได้รับการยืนยันจากผู้ใช้**
- รันบ่อยได้ — จะอัปเดตเฉพาะเมื่อมีข้อมูลใหม่
- `--comprehensive` จะมีปฏิสัมพันธ์กับผู้ใช้เสมอ
