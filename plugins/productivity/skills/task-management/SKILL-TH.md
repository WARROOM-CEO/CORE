# สกิล: การจัดการงาน

**จัดการงานผ่านไฟล์ TASKS.md ที่ทั้งคุณและ Claude สามารถแก้ไขได้**

---

## สกิลนี้ทำอะไร?

ติดตามงานทั้งหมดในไฟล์ `TASKS.md` ไฟล์เดียว — ง่าย อ่านได้ด้วยตาเปล่า และซิงค์กับแผงควบคุมอัตโนมัติ

---

## ที่อยู่ไฟล์

**ใช้ `TASKS.md` ในโฟลเดอร์ทำงานปัจจุบันเสมอ**
- ถ้ามีอยู่แล้ว → อ่านและเขียนทับ
- ถ้ายังไม่มี → สร้างพร้อมเทมเพลตมาตรฐาน

---

## รูปแบบเทมเพลต

```markdown
# รายการงาน

## กำลังดำเนินการ

## รอผู้อื่น

## งานในอนาคต

## เสร็จสมบูรณ์
```

รูปแบบงาน:
- `- [ ] **ชื่องาน** - บริบท สำหรับใคร กำหนดส่ง`
- ข้อย่อยสำหรับรายละเอียดเพิ่มเติม
- เมื่อเสร็จ: `- [x] ~~งาน~~ (วันที่)`

---

## วิธีที่ Claude จะตอบสนอง

| ผู้ใช้พูดว่า | Claude ทำอะไร |
|------------|--------------|
| "งานของฉันมีอะไรบ้าง" | อ่าน TASKS.md และสรุปรายการกำลังดำเนินการ + รอผู้อื่น |
| "เพิ่มงาน..." / "จำไว้ว่าต้อง..." | เพิ่มในกำลังดำเนินการพร้อมบริบท |
| "เสร็จแล้ว X" / "ทำ X เสร็จแล้ว" | ค้นหางาน → เปลี่ยน `[ ]` เป็น `[x]` → ย้ายไปเสร็จสมบูรณ์ |
| "รอใครอยู่บ้าง" | อ่านรอผู้อื่นและแสดงว่าแต่ละรายการรออยู่นานแค่ไหน |

---

## หลักเกณฑ์การเขียนงาน

- **ตัวหนา**ชื่องาน เพื่อสแกนได้ง่าย
- ระบุ "สำหรับ [ชื่อ]" ถ้าเป็นงานที่สัญญากับคนอื่น
- ระบุ "ครบกำหนด [วันที่]" สำหรับวันสุดท้าย
- ระบุ "ตั้งแต่ [วันที่]" สำหรับงานที่รอผู้อื่น
- ข้อย่อยสำหรับบริบทเพิ่มเติม
- ล้างส่วนเสร็จสมบูรณ์ทุกประมาณ 1 สัปดาห์

---

## การดึงงานจากการสนทนา

เมื่อสรุปการประชุมหรือบทสนทนา Claude จะเสนอเพิ่มงานที่พบ เช่น:
- งานที่ผู้ใช้รับปาก ("ฉันจะส่งให้")
- รายการกิจกรรมที่ได้รับมอบหมาย
- การติดตามผลที่พูดถึง

**Claude จะสอบถามก่อนเสมอ — ไม่เพิ่มงานเองโดยไม่ได้รับการยืนยัน**
