# Skill: performance-review — ประเมินผลการทำงาน

**ประเภท**: User-invoked — `/performance-review <ชื่อพนักงาน หรือ review cycle>`

สร้าง templates และโครงสร้างสำหรับ performance review ครอบคลุม self-assessment, manager review และ calibration

---

## วิธีเรียกใช้

```
/performance-review self-assessment
/performance-review manager Sarah Chen
/performance-review calibration Q4 2025
/performance-review [ชื่อพนักงาน หรือ review cycle]
```

---

## 3 โหมด

### โหมด 1: Self-Assessment Template
สำหรับพนักงานที่เตรียม self-assessment ด้วยตนเอง

รูปแบบผลลัพธ์:
```
## Self-Assessment: [รอบการประเมิน]

### ผลงานสำคัญ
[3-5 ผลงานหลัก พร้อม Situation → Contribution → Impact]

### ทบทวนเป้าหมาย
| เป้าหมาย | สถานะ | หลักฐาน |
|---------|-------|---------|

### ด้านที่เติบโต | ความท้าทาย | เป้าหมายรอบต่อไป
```

### โหมด 2: Manager Review Template
สำหรับ manager เขียน review ให้ direct report

รูปแบบผลลัพธ์:
```
## Performance Review: [ชื่อพนักงาน]

### Overall Rating: Exceeds / Meets / Below Expectations

### สรุปผลการทำงาน (2-3 ประโยค)

### จุดแข็งหลัก (พร้อมตัวอย่างเจาะจง)

### ด้านที่ต้องพัฒนา (พร้อม action ที่ทำได้)

### Goal Achievement | Impact | Development Plan | Comp Recommendation
```

### โหมด 3: Calibration Prep
สำหรับ manager เตรียม calibration session กับ leadership

รูปแบบผลลัพธ์:
```
## Calibration Prep: [Review Cycle]

### Team Overview Table (Rating | Level | Tenure | Notes)

### Rating Distribution vs. Company Target (~15-20% Exceeds, ~60-70% Meets)

### Calibration Discussion Points | Promotion Candidates | Comp Actions
```

## เคล็ดลับ Feedback ที่ดี

**เจาะจง** — "งานเอกสารไม่ครบถ้วนในไตรมาสที่ผ่านมา" ดีกว่า "คุณไม่ระมัดระวัง"

**เน้น behaviors ไม่ใช่ personality** — พูดถึงสิ่งที่สังเกตได้ ไม่ใช่ตัวตน

**Balance positive และ constructive** — ทั้งสองส่วนสำคัญเท่ากัน และไม่ควรเป็นเรื่องเซอร์ไพรส์

**Development ต้องทำได้จริง** — "ปรับปรุงการสื่อสาร" คลุมเครือ "นำเสนอใน all-hands ไตรมาสหน้า" ทำได้จริง

## การเชื่อมต่อที่ช่วยเพิ่มประสิทธิภาพ

เมื่อเชื่อมต่อ **~~HRIS** — ดึง review history และ goal tracking จากรอบก่อน

เมื่อเชื่อมต่อ **~~project tracker** — ดึง completed work และ contributions เพื่อใช้เป็นหลักฐานใน review
