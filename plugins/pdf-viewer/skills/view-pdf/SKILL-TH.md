# view-pdf — ตัวแสดงผล PDF แบบโต้ตอบ

**สกิลนี้ทำงานอัตโนมัติเมื่อคุณต้องการดู ทำงานร่วม หรืออธิบายประกอบเอกสาร PDF**

---

## สกิลนี้ทำอะไร?

ให้คุณเข้าถึง PDF server ในเครื่องที่แสดงเอกสารในตัวแสดงผลสด พร้อมรองรับ:

- เปิดและแสดง PDF จาก local หรือ URL
- อธิบายประกอบแบบร่วมมือ — Claude เสนอ markup ทีละส่วน คุณตรวจสอบ
- กรอกแบบฟอร์มพร้อม visual feedback แบบ real-time
- วางลายเซ็นหรืออักษรย่อเป็นรูปภาพ
- ประทับตรา เช่น APPROVED, DRAFT, CONFIDENTIAL

---

|------------|------------|
| "แสดงสัญญานี้ให้ดูหน่อย" | เปิด PDF ในตัวแสดงผล |

| "เซ็นหน้า 3" / "เพิ่มอักษรย่อทุกหน้า" | วางรูปลายเซ็นที่ตำแหน่งที่ระบุ |
| "ประทับตรา CONFIDENTIAL" | เพิ่ม stamp annotation |

## เมื่อไหร่ที่ไม่ใช้สกิลนี้

| คุณพูดว่า | Claude จะทำแทน |
|---------|--------------|
| "สรุป PDF นี้ให้หน่อย" | อ่าน PDF โดยตรง (เร็วกว่า) |
| "หน้า 5 พูดว่าอะไร?" | ใช้ Read tool |
| "ดึงตารางจากส่วนที่ 3" | ใช้ Read tool |

ตัวแสดงผลมีคุณค่าเมื่อคุณต้องการ **เห็น** เอกสารและ **ร่วมกันทำเครื่องหมาย** — ไม่ใช่สำหรับการดึงข้อมูลเพียงอย่างเดียว

## เครื่องมือที่รองรับ

### `list_pdfs`
แสดงรายการ PDF local ที่มีและ directory ที่อนุญาต ไม่ต้องมี argument

### `display_pdf`
เปิด PDF ในตัวแสดงผล เรียกหนึ่งครั้งต่อเอกสาร
- รับ local file path หรือ HTTPS URL
- ส่งคืน `viewUUID` สำหรับการเรียก `interact` ทุกครั้งถัดไป
- ส่งคืน `formFields` ถ้า PDF มีช่องกรอก

### `interact`
การทำงานทั้งหมดหลังจาก `display_pdf` ส่ง `viewUUID` พร้อม command ที่ต้องการ

**Annotation:** `add_annotations`, `update_annotations`, `remove_annotations`, `highlight_text`

**Navigation:** `navigate`, `search`, `zoom`

**Extraction:** `get_text` (สูงสุด 20 หน้า), `get_screenshot`

**Form:** `fill_form`

## ประเภทการอธิบายประกอบ

| ประเภท | ใช้สำหรับ |
|--------|---------|
| `highlight` | ทำเครื่องหมายข้อความสำคัญ |
| `underline` | ขีดเส้นใต้คำสำคัญ |
| `strikethrough` | ทำเครื่องหมายส่วนที่ถูกลบ |
| `note` | Sticky note comment |
| `freetext` | ข้อความที่แสดงบนหน้า |
| `rectangle` / `circle` / `line` | รูปทรงเรขาคณิต |
| `stamp` | APPROVED, DRAFT, CONFIDENTIAL ฯลฯ |
| `image` | **ลายเซ็น**, อักษรย่อ, โลโก้ |

## Workflow หลัก

### อธิบายประกอบแบบร่วมมือ
1. เปิด PDF → อ่านเนื้อหาที่เกี่ยวข้อง → เสนอชุด annotations → รอการอนุมัติ → ใช้และถ่าย screenshot → ให้คุณตรวจสอบ → ดำเนินการส่วนถัดไป

### กรอกแบบฟอร์ม
1. เปิด PDF → ตรวจสอบ `formFields` → ถ้าชื่อช่องเข้าใจยาก ให้ screenshot เพื่อดู label จริง → ถามค่าที่ต้องการ → กรอกและแสดงผล

### เซ็นชื่อ
1. ขอ path รูปลายเซ็น → เปิด PDF → หาตำแหน่ง → วางรูป → ยืนยันด้วย screenshot

---

## แหล่ง PDF ที่รองรับ

- ไฟล์ local (path ในเครื่อง)
- arXiv (`/abs/` URL แปลงอัตโนมัติ)
- HTTPS PDF URL ตรง (bioRxiv, Zenodo, OSF ฯลฯ)

---

## ไม่รองรับ

- สรุปหรือดึงข้อความ → ใช้ native Read tool
- ลายเซ็นดิจิทัลที่รับรองทางกฎหมาย → ใช้บริการ signing โดยเฉพาะ
- สร้าง PDF ใหม่ → สกิลนี้ทำงานกับ PDF ที่มีอยู่เท่านั้น
