# อธิบายประกอบ PDF (`/pdf-viewer:annotate`)

**อธิบายประกอบ PDF แบบร่วมมือกัน — เสนอ markup ตรวจสอบด้วยกัน และปรับปรุงซ้ำ**

---

## คำสั่งนี้ทำอะไร?

เดินทางผ่านเอกสารร่วมกับคุณ โดย Claude จะเสนอและใช้การอธิบายประกอบทีละส่วน คุณตรวจสอบแต่ละชุดใน viewer สด ก่อนที่จะดำเนินการต่อ

---

## วิธีใช้

```
/pdf-viewer:annotate
/pdf-viewer:annotate [path หรือ URL ของไฟล์ PDF]
```

---

## ขั้นตอนการทำงาน (โหมด AI-driven — ค่าเริ่มต้น)

1. **เปิด** — แสดง PDF ในตัวแสดงผลสด
2. **ทำความเข้าใจ** — อ่านเนื้อหาในช่วงหน้าที่เกี่ยวข้อง
3. **เสนอ** — Claude อธิบายสิ่งที่จะ annotate เช่น "ฉันจะ highlight ข้อกำหนดการยกเลิกในหน้า 2 เพิ่มหมายเหตุข้างๆ และประทับตรา DRAFT ในหน้า 1 โอเคไหม?"
4. **ใช้** — เมื่อได้รับอนุมัติ เพิ่ม annotations และถ่ายภาพหน้าที่แก้ไข
5. **ตรวจสอบ** — แสดง screenshot และถามว่าต้องการแก้ไขอะไร
6. **ปรับปรุงซ้ำ** — ดำเนินการส่วนถัดไปต่อ
7. **เสร็จสิ้น** — แจ้งให้ download PDF ที่อธิบายประกอบแล้วจาก toolbar

---

## โหมด Manual

หากคุณให้คำสั่งชัดเจน เช่น "highlight ย่อหน้าที่ 3" หรือ "ประทับตรา CONFIDENTIAL ทุกหน้า" Claude จะข้ามขั้นตอนเสนอและดำเนินการทันที ยังคงยืนยันด้วย screenshot เสมอ

---

## ประเภทการอธิบายประกอบที่รองรับ

| ประเภท | คำอธิบาย |
|--------|---------|
| **Highlight** | ทำเครื่องหมายข้อความสำคัญ |
| **Underline** | ขีดเส้นใต้คำสำคัญ |
| **Strikethrough** | ทำเครื่องหมายส่วนที่ถูกลบ |
| **Note** | sticky note comment |
| **Freetext** | ข้อความที่แสดงบนหน้า |
| **Rectangle / Circle / Line** | รูปทรงเรขาคณิต |
| **Stamp** | ตราประทับ เช่น APPROVED, DRAFT, CONFIDENTIAL |
| **Image** | ลายเซ็น, อักษรย่อ, โลโก้ |

---

## เคล็ดลับ

- **ให้ใช้ `highlight_text` แทนการกำหนด coordinates เอง** — Claude จะหาตำแหน่งของข้อความให้อัตโนมัติ
- **รวม annotations ที่เกี่ยวข้องไว้ใน batch เดียวกัน** — เร็วกว่าและตรวจสอบง่ายกว่า
- **ทุก batch ลงท้ายด้วย screenshot** — คุณเห็นผลลัพธ์ก่อนดำเนินการต่อ
- **เสนอทีละ 3-5 annotations** — ตรวจสอบง่าย ปรับแก้ได้สะดวก
