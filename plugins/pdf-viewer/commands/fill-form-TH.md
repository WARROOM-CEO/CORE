# กรอกแบบฟอร์ม PDF (`/pdf-viewer:fill-form`)

**กรอกช่องแบบฟอร์ม PDF แบบโต้ตอบพร้อม visual feedback สด**

---

## คำสั่งนี้ทำอะไร?

ช่วยคุณกรอกแบบฟอร์ม PDF ในตัวแสดงผลสด โดยให้ feedback ทางภาพแบบ real-time ทุกช่องที่กรอก ต่างจากเครื่องมือกรอกแบบฟอร์มแบบ programmatic ทั่วไปที่คุณเห็นเพียงข้อความยืนยันความสำเร็จ

---

## วิธีใช้

```
/pdf-viewer:fill-form
/pdf-viewer:fill-form [path หรือ URL ของไฟล์ PDF]
```

---

## ทำไมต้องใช้คำสั่งนี้แทนการกรอกแบบ programmatic?

- **ยืนยันด้วยภาพ** — คุณเห็นค่าแต่ละตัวลงในช่องที่ถูกต้อง ไม่ใช่แค่ข้อความว่า "สำเร็จ"
- **ช่องที่ไม่มีชื่อ** — PDF จำนวนมากมีช่องที่มีชื่อแบบ machine เช่น `Text1`, `Field_7` โดย label จริงอย่าง "วันเกิด" หรือ "เลขบัตร" จะพิมพ์อยู่ข้างๆ ช่องบนหน้า Claude จะใช้ screenshot เพื่อดูว่าแต่ละช่องคืออะไร
- **แก้ไขได้ง่าย** — คุณสามารถแก้ไขหรือล้างช่องใดก็ได้โดยตรงใน viewer

---

## สองแนวทาง

### 1. User-driven (แบบฟอร์มที่มี label ชัดเจน)

เปิด PDF พร้อมตัวเลือก `elicit_form_inputs: true` — ระบบตรวจจับช่องแบบฟอร์มและถามคุณก่อนเปิด viewer จากนั้นแสดง PDF ที่กรอกแล้ว

### 2. AI-assisted (แบบฟอร์มซับซ้อน ช่องไม่มีชื่อ)

1. เปิด PDF และตรวจสอบ `formFields` (ชื่อ, ประเภท, หน้า, bounding box)
2. ถ้าชื่อช่องเข้าใจยาก (`Text1`, `Field_7`) Claude จะ screenshot แต่ละหน้าและจับคู่กับ label ที่มองเห็น
3. ถามค่าที่ต้องการโดยอ้างอิง label ภาพ เช่น "ช่อง 'วันเกิด' ในหน้า 1"
4. กรอกข้อมูลและแสดง screenshot ให้ยืนยัน

---

## ตัวอย่าง

```
ช่วยกรอกแบบฟอร์ม W-9 นี้ให้หน่อย
```

Claude จะ: เปิด PDF → ตรวจสอบชื่อช่อง → screenshot เพื่อดู label จริง → ถามข้อมูลที่ยังขาด → กรอกทุกช่อง → แสดง screenshot เพื่อยืนยัน

---

## หมายเหตุ

- ช่องลายเซ็นมักเป็นแยกต่างหาก — กรอกข้อความก่อน แล้วใช้ `/pdf-viewer:sign` สำหรับลายเซ็น
- ค่า Checkbox/radio ใช้ `true`/`false` หรือ string ของตัวเลือก
- คุณสามารถลากและแก้ไขช่องโดยตรงใน viewer ได้เสมอ
