# การสรุปข้อมูลการประชุม — จัดทำเอกสารสรุปข้อมูลการประชุมด้านกฎหมาย

**สกิลอ้างอิงภายใน — Claude ใช้เมื่อเตรียมการประชุมที่มีความเกี่ยวข้องด้านกฎหมาย**

> สกิลนี้เป็นแบบ Model-invoked — Claude จะเรียกใช้อัตโนมัติเมื่อเตรียมการประชุมที่มีบริบทด้านกฎหมาย

> **หมายเหตุสำคัญ**: ช่วยในการทำงานด้านกฎหมายแต่ไม่ใช่การให้คำแนะนำทางกฎหมาย เอกสารสรุปควรได้รับการตรวจสอบก่อนนำไปใช้

---

## สกิลนี้ทำอะไร?

รวบรวมข้อมูลจากแหล่งที่เชื่อมต่อ จัดทำเอกสารสรุปข้อมูลที่มีโครงสร้างสำหรับการประชุมที่มีความเกี่ยวข้องด้านกฎหมาย และติดตามรายการกิจกรรมที่เกิดขึ้น

---

## ประเภทการประชุมและสิ่งที่ต้องเตรียม

| ประเภทการประชุม | สิ่งที่ต้องเตรียม |
|---------------|----------------|
| **การทบทวนข้อตกลง** | สถานะสัญญา ประเด็นที่ค้างอยู่ ประวัติของคู่สัญญา กลยุทธ์การเจรจา การอนุมัติที่ต้องการ |
| **คณะกรรมการ / คณะทำงาน** | รายงานอัปเดตด้านกฎหมาย ประเด็นสำคัญในทะเบียนความเสี่ยง การพัฒนาด้านกฎระเบียบ ร่างมติ |
| **การประชุมกับผู้ขาย** | สถานะสัญญา ประเด็นที่ค้างอยู่ มาตรฐานการดำเนินงาน เป้าหมายการเจรจา |
| **การประสานงานทีม** | สถานะภาระงาน เรื่องสำคัญ กำหนดเวลาที่กำลังจะมาถึง |
| **หน่วยงานกำกับดูแล / รัฐบาล** | ประวัติของเรื่อง สถานะการปฏิบัติตามกฎระเบียบ การสื่อสารก่อนหน้า เอกสารสรุปสำหรับทนายความ |
| **การฟ้องร้อง / ข้อพิพาท** | สถานะคดี การพัฒนาล่าสุด เงื่อนไขการไกล่เกลี่ย |

## แหล่งที่ดึงข้อมูล

| แหล่ง | ข้อมูลที่ดึง |
|-------|------------|
| **ปฏิทิน** | รายละเอียดการประชุม การประชุมก่อนหน้ากับผู้เข้าร่วมเดิม |
| **อีเมล** | การสื่อสารล่าสุดกับหรือเกี่ยวกับผู้เข้าร่วม รายการกิจกรรมที่ค้างอยู่ |
| **แชท** | การสนทนาล่าสุดเกี่ยวกับหัวข้อการประชุม |
| **เอกสาร** | วาระการประชุม บันทึกการประชุมก่อนหน้า สัญญาที่เกี่ยวข้อง |
| **ระบบจัดการสัญญา** | สัญญากับคู่สัญญา สถานะการเจรจา |
| **ระบบจัดการลูกค้าสัมพันธ์** | ข้อมูลบัญชี ขั้นตอนข้อตกลง แผนผังผู้มีส่วนได้ส่วนเสีย |

## โครงสร้างเอกสารสรุปการประชุมที่ได้รับ

```
รายละเอียดการประชุม — ชื่อ เวลา ระยะเวลา บทบาทของคุณ
ผู้เข้าร่วม — ชื่อ บทบาท ความสนใจ บันทึกเพิ่มเติม
วาระการประชุม — หัวข้อที่คาดว่าจะพูดถึง
พื้นหลังและบริบท — สรุป 2-3 ย่อหน้า
เอกสารสำคัญ — เอกสารสำคัญและที่เก็บรักษา
ประเด็นที่ค้างอยู่ — ตารางปัญหาที่ค้างอยู่พร้อมผู้รับผิดชอบ
ประเด็นทางกฎหมาย — ประเด็นด้านกฎหมายเฉพาะเจาะจง
ประเด็นสำคัญ — 3+ ประเด็นหลักพร้อมบริบท
คำถามที่ต้องถาม — คำถามที่ต้องสอบถาม
การตัดสินใจที่ต้องการ — การตัดสินใจพร้อมตัวเลือก
จุดที่ไม่สามารถยอมได้ — จุดที่ไม่สามารถยอมแพ้ได้ (สำหรับการเจรจา)
การติดตามผลการประชุมก่อนหน้า — รายการกิจกรรมที่ค้างจากการประชุมก่อนหน้า
ช่องว่างการเตรียมการ — ข้อมูลที่หาไม่ได้
```

## การติดตามรายการกิจกรรม

### ระหว่างหรือหลังการประชุม
Claude จะช่วยรวบรวมรายการกิจกรรมในรูปแบบ:

| # | รายการกิจกรรม | ผู้รับผิดชอบ | กำหนดเวลา | ลำดับความสำคัญ | สถานะ |
|---|-------------|-------|----------|----------|--------|

**หลักการของรายการกิจกรรม:**
- **เฉพาะเจาะจง** — "ส่งข้อเสนอแก้ไขของข้อความ 4.2 ไปยังทนายความของคู่สัญญา" ไม่ใช่ "ติดตามสัญญา"
- **มีผู้รับผิดชอบคนเดียว** — ทุกรายการกิจกรรมต้องมีผู้รับผิดชอบหนึ่งคน
- **มีกำหนดเวลาชัดเจน** — วันที่เฉพาะเจาะจง ไม่ใช่ "เร็วๆ นี้"

### ความถี่ในการติดตาม
- **ลำดับความสำคัญสูง** — ตรวจสอบทุกวันจนเสร็จสมบูรณ์
- **ลำดับความสำคัญปานกลาง** — ตรวจสอบในการประชุมประสานงานทีมหรือการทบทวนรายสัปดาห์
- **ลำดับความสำคัญต่ำ** — ตรวจสอบในการประชุมครั้งต่อไปหรือรายเดือน
- **รายการที่เลยกำหนดเวลา** — แจ้งเตือนไปยังผู้รับผิดชอบและผู้จัดการ

### การติดตามผลหลังการประชุม
1. แจกจ่ายรายการกิจกรรมให้ผู้เข้าร่วมทุกคน
2. ตั้งการแจ้งเตือนในปฏิทินสำหรับกำหนดเวลา
3. อัปเดตระบบที่เกี่ยวข้อง (ระบบจัดการสัญญา ระบบจัดการคดี ทะเบียนความเสี่ยง)
4. จัดเก็บบันทึกการประชุมในที่เก็บเอกสาร
5. ระบุรายการเร่งด่วนที่ต้องการความสนใจทันที
