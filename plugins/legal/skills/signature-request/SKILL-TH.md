# การขอลงลายเซ็น — ส่งเอกสารเพื่อลงลายเซ็น

**เตรียมและส่งเอกสารเพื่อลงลายเซ็นอิเล็กทรอนิกส์ พร้อมรายการตรวจสอบก่อนลงลายเซ็นและการกำหนดลำดับการลงลายเซ็น**

---

## สกิลนี้ทำอะไร?

เตรียมเอกสารสำหรับการลงลายเซ็นอิเล็กทรอนิกส์ — ตรวจสอบความสมบูรณ์ กำหนดลำดับการลงลายเซ็น และส่งเพื่อดำเนินการ

> **หมายเหตุสำคัญ**: ช่วยในการทำงานด้านกฎหมายแต่ไม่ใช่การให้คำแนะนำทางกฎหมาย ตรวจสอบว่าเอกสารอยู่ในรูปแบบสุดท้ายก่อนส่งเพื่อลงลายเซ็น

---

## วิธีใช้

```
/signature-request
/signature-request <เอกสารหรือสัญญาที่ต้องการส่ง>
```

**รูปแบบที่รับได้:** PDF, DOCX, URL (ไปยังระบบจัดเก็บบนคลาวด์หรือระบบจัดการสัญญา) หรืออ้างอิง เช่น "สัญญากรอบบริการของบริษัท Acme Corp ที่สรุปเมื่อวาน"

---

## ขั้นตอนการทำงาน

### 1. รับเอกสาร
รับเอกสารในรูปแบบที่รองรับ

### 2. รายการตรวจสอบก่อนลงลายเซ็น

```
รายการตรวจสอบก่อนลงลายเซ็น

□ เอกสารอยู่ในรูปแบบสุดท้ายที่ตกลงแล้ว (ไม่มีข้อเสนอแก้ไขที่ค้างอยู่)
□ เอกสารแนบและภาคผนวกทั้งหมดแนบครบถ้วน
□ ชื่อนิติบุคคลถูกต้องในช่องลงลายเซ็น
□ วันที่ถูกต้องหรือเว้นว่างสำหรับวันที่ดำเนินการ
□ ช่องลงลายเซ็นตรงกับผู้มีอำนาจลงนาม
□ ได้รับการอนุมัติภายในที่จำเป็นเรียบร้อยแล้ว
□ เอกสารได้รับการตรวจสอบโดยทนายความที่เหมาะสม
```

### 3. กำหนดการลงลายเซ็น
Claude จะสอบถาม:
- **ผู้ลงลายเซ็น** — ชื่อ อีเมล ตำแหน่ง
- **ลำดับการลงลายเซ็น** — ตามลำดับหรือพร้อมกัน?
- **การอนุมัติภายใน** — มีใครต้องอนุมัติก่อนที่คู่สัญญาจะลงลายเซ็นหรือไม่?
- **ผู้รับสำเนา** — ใครควรได้รับสำเนาเอกสารที่ลงลายเซ็นแล้ว?

### 4. ส่งเพื่อลงลายเซ็น

**ถ้าเชื่อมต่อกับระบบลายเซ็นอิเล็กทรอนิกส์ (DocuSign, Adobe Sign ฯลฯ):**
- สร้างชุดเอกสารสำหรับลงลายเซ็น
- กำหนดช่องลงลายเซ็นและลำดับ
- เพิ่มช่องลงชื่อย่อหรือช่องวันที่ที่จำเป็น
- ส่งเพื่อลงลายเซ็น

**ถ้าไม่ได้เชื่อมต่อ:**
- สร้างเอกสารคำแนะนำการลงลายเซ็น
- แสดงรายการผู้ลงลายเซ็นทั้งหมดพร้อมข้อมูลติดต่อ

## ผลลัพธ์ที่จะได้รับ

```
การขอลงลายเซ็น: [ชื่อเอกสาร]

รายละเอียดเอกสาร — ประเภท คู่สัญญา จำนวนหน้า
การตรวจสอบก่อนลงลายเซ็น — ผ่าน / พบปัญหา
การกำหนดการลงลายเซ็น — ตารางผู้ลงลายเซ็นพร้อมลำดับและบทบาท
ผู้รับสำเนา
สถานะ — ส่งแล้ว / พร้อม / ปัญหาที่ต้องแก้ไข
ขั้นตอนต่อไป — สิ่งที่คาดหวังหลังส่ง ระยะเวลาที่คาดว่าจะได้รับ
```

## เคล็ดลับ

1. **ตรวจสอบชื่อนิติบุคคลอย่างระมัดระวัง** — ข้อผิดพลาดที่พบบ่อยที่สุดในการลงลายเซ็นคือชื่อนิติบุคคลที่ไม่ถูกต้อง
2. **ตรวจสอบอำนาจ** — ตรวจสอบว่าผู้ลงลายเซ็นแต่ละคนมีอำนาจผูกพันองค์กรของตน
3. **เก็บสำเนาไว้** — ควรจัดเก็บเอกสารที่ลงลายเซ็นแล้วในระบบจัดเก็บบนคลาวด์หรือระบบจัดการสัญญาทันทีหลังดำเนินการเสร็จสิ้น
