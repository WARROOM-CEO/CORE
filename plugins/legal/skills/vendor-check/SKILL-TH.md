# การตรวจสอบสถานะผู้ขาย — ตรวจสอบสถานะสัญญากับผู้ขาย

**ตรวจสอบสถานะสัญญาที่มีอยู่กับผู้ขายทั่วทุกระบบที่เชื่อมต่อ พร้อมการวิเคราะห์ช่องว่างและกำหนดเวลาที่กำลังจะมาถึง**

---

## สกิลนี้ทำอะไร?

ตรวจสอบสถานะสัญญาที่มีอยู่กับผู้ขายทั่วทุกระบบที่เชื่อมต่อ สร้างมุมมองรวมของความสัมพันธ์ทางกฎหมาย พร้อมการวิเคราะห์ช่องว่าง

> **หมายเหตุสำคัญ**: ช่วยในการทำงานด้านกฎหมายแต่ไม่ใช่การให้คำแนะนำทางกฎหมาย ควรตรวจสอบกับเอกสารต้นฉบับโดยผู้เชี่ยวชาญ

---

## วิธีใช้

```
/vendor-check
/vendor-check [ชื่อผู้ขาย]
```

**ตัวอย่าง:** `/vendor-check บริษัท Acme Corp`, `/vendor-check AWS`, `/vendor-check Google`

---

## ระบบที่ค้นหา (ตามลำดับความสำคัญ)

| ระบบ | สิ่งที่ค้นหา |
|------|------------|
| **ระบบจัดการสัญญา** | สัญญาที่ใช้งานอยู่ สัญญาที่หมดอายุ (3 ปีย้อนหลัง) กำลังเจรจา รอลงลายเซ็น ข้อเสนอแก้ไข |
| **ระบบจัดการลูกค้าสัมพันธ์** | สถานะบัญชี ข้อมูลติดต่อของทีมกฎหมายผู้ขาย |
| **อีเมล** | การสื่อสารที่เกี่ยวกับสัญญา (6 เดือนล่าสุด) ข้อตกลงความลับหรือสัญญาที่แนบมา |
| **เอกสาร** | สัญญาที่ลงลายเซ็นแล้ว ข้อเสนอแก้ไข การตรวจสอบเนื้อหา |
| **แชท** | การกล่าวถึงล่าสุดเกี่ยวกับผู้ขาย (3 เดือนล่าสุด) |

## การวิเคราะห์ช่องว่างที่จะได้รับ

```
การครอบคลุมสัญญา

✅ ข้อตกลงความลับ — [สถานะ]
✅/❌ สัญญากรอบบริการ — [สถานะ หรือ "ไม่พบ"]
✅/❌ สัญญาการคุ้มครองข้อมูล — [สถานะ หรือ "ไม่พบ"]
✅/❌ ใบสั่งงาน — [สถานะ หรือ "ไม่พบ"]
✅/❌ ข้อตกลงระดับการให้บริการ — [สถานะ หรือ "ไม่พบ"]
✅/❌ ใบรับรองประกันภัย — [สถานะ หรือ "ไม่พบ"]
```

**การแจ้งเตือนอัตโนมัติ:** ถ้ามีสัญญากรอบบริการแต่ไม่มีสัญญาการคุ้มครองข้อมูล และผู้ขายจัดการข้อมูลส่วนบุคคล

## รายงานที่จะได้รับ

```
สถานะสัญญากับผู้ขาย: [ชื่อผู้ขาย]
วันที่ค้นหา: [วันที่]
แหล่งที่ตรวจสอบ: [รายการระบบที่ค้นหา]
แหล่งที่ไม่พร้อมใช้งาน: [ระบบที่ไม่ได้เชื่อมต่อ]

ภาพรวมความสัมพันธ์
  ผู้ขาย: [ชื่อนิติบุคคลเต็ม]
  ประเภทความสัมพันธ์: [ผู้ขาย/พันธมิตร/ลูกค้า]
  สถานะในระบบ CRM: [ถ้ามี]

สรุปสัญญา
  [ประเภทสัญญา] — [สถานะ]
  - มีผลใช้บังคับ: [วันที่]
  - หมดอายุ: [วันที่] (ต่ออายุอัตโนมัติ / ไม่ต่ออายุอัตโนมัติ)
  - ข้อกำหนดสำคัญ: [สรุป]
  - ที่เก็บ: [ที่เก็บ]

การวิเคราะห์ช่องว่าง — สิ่งที่มีและสิ่งที่อาจต้องการ

กิจกรรมที่กำลังจะมาถึง
  - สัญญาที่จะหมดอายุเร็วๆ นี้
  - สัญญาที่ยังไม่มีแต่จำเป็น
  - ข้อเสนอแก้ไขที่อาจต้องการ
```

## หมายเหตุสำคัญ

- ถ้าไม่พบสัญญาในระบบที่เชื่อมต่อ — Claude จะสอบถามว่ามีสัญญาเก็บไว้ที่อื่นหรือไม่
- สำหรับกลุ่มผู้ขาย (มีบริษัทลูก) — Claude จะสอบถามว่าต้องการตรวจสอบนิติบุคคลเฉพาะเจาะจงหรือทั้งกลุ่ม
- **การแจ้งเตือนอัตโนมัติ:** สัญญาที่หมดอายุแล้วแต่ยังมีภาระผูกพันที่คงอยู่ (ข้อตกลงความลับ การชดใช้ค่าเสียหาย)
- **การแจ้งเตือนอัตโนมัติ:** สัญญาที่จะหมดอายุภายใน 90 วัน — แสดงอย่างชัดเจน
