# การตอบกลับด้านกฎหมาย — สร้างการตอบกลับคำถามด้านกฎหมายจากเทมเพลต

**สร้างการตอบกลับคำถามด้านกฎหมายทั่วไปโดยใช้เทมเพลตที่กำหนดไว้ พร้อมตรวจสอบสัญญาณเตือนการแจ้งระดับสูงอัตโนมัติ**

---

## สกิลนี้ทำอะไร?

สร้างการตอบกลับสำหรับคำถามด้านกฎหมายทั่วไปโดยใช้เทมเพลตที่กำหนดไว้ พร้อมตรวจสอบสัญญาณเตือนสำหรับสถานการณ์ที่ไม่ควรใช้การตอบกลับแบบเทมเพลต

> **หมายเหตุสำคัญ**: การตอบกลับที่สร้างขึ้นควรได้รับการตรวจสอบโดยผู้เชี่ยวชาญด้านกฎหมายที่มีคุณวุฒิก่อนส่ง โดยเฉพาะสำหรับการสื่อสารที่อยู่ภายใต้การควบคุม

---

## วิธีใช้

```
/legal-response
/legal-response [ประเภทคำถาม]
```

**ประเภทคำถามที่รองรับ:**
- `dsr` / `คำขอเจ้าของข้อมูล` — คำขอเข้าถึง/ลบ/แก้ไขข้อมูลส่วนบุคคล
- `hold` / `คำสั่งระงับการทำลายหลักฐาน` — ประกาศคำสั่งระงับการทำลายหลักฐาน
- `vendor` / `คำถามจากผู้ขาย` — คำถามด้านกฎหมายจากผู้ขาย
- `nda` / `คำขอข้อตกลงความลับ` — คำขอข้อตกลงความลับจากทีมธุรกิจ
- `privacy` / `คำถามด้านความเป็นส่วนตัว` — คำถามเกี่ยวกับความเป็นส่วนตัว
- `subpoena` — การตอบคำสั่งศาลหรือกระบวนการทางกฎหมาย
- `insurance` — การแจ้งเคลมประกันภัย
- `custom` — ใช้เทมเพลตที่กำหนดเอง

## ขั้นตอนการทำงาน

### 1. ระบุประเภทคำถาม
รับประเภทจากคุณ ถ้าไม่ชัดเจนจะแสดงหมวดหมู่ให้เลือก

### 2. โหลดเทมเพลต
ค้นหาเทมเพลตจากไฟล์การตั้งค่าท้องถิ่น (`legal.local.md`) ถ้าไม่มีจะเสนอช่วยสร้างเทมเพลตใหม่

### 3. ตรวจสอบสัญญาณเตือนการแจ้งระดับสูง

**Claude จะหยุดทันทีและไม่ใช้เทมเพลตถ้าพบ:**
- เรื่องที่เกี่ยวข้องกับการฟ้องร้องหรือการสอบสวนของหน่วยงานกำกับดูแล
- คำถามจากหน่วยงานรัฐบาลหรือหน่วยงานบังคับใช้กฎหมาย
- การตอบกลับที่อาจสร้างข้อผูกพันทางกฎหมายหรือการสละสิทธิ์
- เรื่องที่เกี่ยวข้องกับความรับผิดทางอาญา
- ความสนใจจากสื่อมวลชน
- หลายเขตอำนาจที่มีข้อกำหนดขัดแย้งกัน
- เรื่องที่เกี่ยวข้องกับผู้บริหารระดับสูงหรือคณะกรรมการ

เมื่อพบสัญญาณเตือนการแจ้งระดับสูง Claude จะ: หยุด → แจ้งเตือน → อธิบายสัญญาณเตือนที่พบ → แนะนำเส้นทางการแจ้งเตือน → เสนอร่างเอกสารสำหรับทนายความตรวจสอบ (ใส่เครื่องหมาย "ฉบับร่าง - สำหรับทนายความตรวจสอบเท่านั้น")

### 4. รวบรวมข้อมูลเฉพาะ
สอบถามรายละเอียดที่จำเป็นตามประเภทคำถาม (ชื่อ วันที่ กฎระเบียบที่บังคับใช้ กำหนดเวลา ฯลฯ)

### 5. สร้างการตอบกลับ
ใส่ข้อมูลลงในเทมเพลตพร้อม: น้ำเสียงที่เหมาะสม ส่วนประกอบทางกฎหมายที่จำเป็น วันที่และกำหนดเวลา ขั้นตอนต่อไปที่ชัดเจน

## ประเภทการตอบกลับ

| ประเภท | ประเภทย่อยหลัก |
|--------|-----------------|
| **คำขอเจ้าของข้อมูล** | การรับทราบ การยืนยันตัวตน การดำเนินการตามคำขอ การปฏิเสธ การขยายเวลา |
| **คำสั่งระงับการทำลายหลักฐาน** | ประกาศครั้งแรก เตือนความจำ การเปลี่ยนแปลงขอบเขต การปลดระงับ |
| **คำถามด้านความเป็นส่วนตัว** | คุกกี้/การติดตาม นโยบายความเป็นส่วนตัว การแบ่งปันข้อมูล การโอนข้ามพรมแดน |
| **คำถามจากผู้ขาย** | สถานะสัญญา คำขอแก้ไข เพิ่มเติม การตอบกลับการตรวจสอบ |
| **คำขอข้อตกลงความลับ** | ส่งแบบฟอร์มมาตรฐาน รับพร้อมข้อเสนอแก้ไข ปฏิเสธพร้อมคำอธิบาย |
| **คำสั่งศาล** | การรับทราบ การคัดค้าน คำขอขยายเวลา (⚠️ ต้องผ่านทนายความเสมอ) |
| **ประกันภัย** | การแจ้งเคลมครั้งแรก ข้อมูลเพิ่มเติม การตอบกลับเกี่ยวกับการสงวนสิทธิ์ |

---

## การจัดการเทมเพลต

เทมเพลตที่ดีควรมี: ประเภท ชื่อ กรณีการใช้งาน สัญญาณเตือน ตัวแปรที่จำเป็น เนื้อหาหลัก กิจกรรมติดตามผล และวันที่ตรวจสอบล่าสุด

Claude จะเสนออัปเดตเทมเพลตเมื่อคุณแก้ไขการตอบกลับระหว่างใช้งาน เพื่อปรับปรุงเทมเพลตในอนาคต
