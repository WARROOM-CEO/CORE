# Roadmap Update — บริหาร Product Roadmap

**ประเภท**: User-invoked (slash command)

**Trigger**: `/roadmap-update <update description>`

## ที่เก็บข้อมูล

อัปเดต สร้าง หรือปรับลำดับความสำคัญ product roadmap

## คุณสมบัติหลัก

- **Status Overview** — Items in progress, completed, at risk
- **Roadmap Items** — Name, description, status, timeframe, owner, dependencies
- **Prioritization Frameworks** — RICE, MoSCoW, ICE, Value vs Effort matrix
- **Dependency Mapping** — Technical, team, external, knowledge dependencies
- **Capacity Planning** — 70% features, 20% tech health, 10% unplanned buffer
- **Roadmap Formats** — Now/Next/Later, Quarterly Themes, OKR-Aligned, Timeline/Gantt

## Output Format

Clear, scannable format. Tables for items. Status labels: Done, On Track, At Risk, Blocked, Not Started.

## ใช้เมื่อ

- Add new initiative และตัดสินใจว่าอะไรต้องขยับออก
- Shift priorities หลังได้ข้อมูลใหม่
- Move timelines due to dependency slip
- Build Now/Next/Later view ตั้งแต่ต้น

## เชื่อมโยงกับ

- Project tracker (pull roadmap items, update statuses)
- Chat (share roadmap changes with team)
- Calendar (account for PTO affecting capacity)
