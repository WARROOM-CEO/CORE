# Write Spec — เขียน Feature Spec / PRD

**ประเภท**: User-invoked (slash command)

**Trigger**: `/write-spec <feature or problem statement>`

## ที่เก็บข้อมูล

เขียน feature spec หรือ PRD (Product Requirements Document) จาก problem statement หรือ feature idea

## คุณสมบัติหลัก

- **Problem Statement** — User problem, who is affected, impact of not solving
- **Goals & Non-Goals** — 3-5 specific outcomes, 3-5 things explicitly out of scope
- **User Stories** — Standard format grouped by persona
- **Requirements** — Categorized as P0 (Must-Have), P1 (Nice-to-Have), P2 (Future)
- **Acceptance Criteria** — Given/When/Then format or checklist
- **Success Metrics** — Leading and lagging indicators with specific targets
- **Open Questions** — Unresolved questions tagged with who needs to answer
- **Scope Management** — Prevent scope creep with tight P0 definitions

## Output Format

Markdown with clear headers. Scannable format for stakeholders to read headers + bold text.

## ใช้เมื่อ

- Turn vague idea หรือ user request เป็น structured document
- Scope feature with goals, non-goals, success metrics
- Define acceptance criteria
- Break big ask into phased specs

## เชื่อมโยงกับ

- Project tracker (search related tickets, dependencies)
- Knowledge base (search prior specs, design docs, research)
- Design (pull mockups, wireframes, design explorations)
