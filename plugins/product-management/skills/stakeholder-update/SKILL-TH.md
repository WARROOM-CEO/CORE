# Stakeholder Update — สร้าง Status Update

**ประเภท**: User-invoked (slash command)

**Trigger**: `/stakeholder-update <update type and audience>`

## ที่เก็บข้อมูล

สร้าง stakeholder update ที่เหมาะกับผู้รับและความถี่

## คุณสมบัติหลัก

- **Update Templates** — Executive, Engineering, Cross-functional, Customer, Board
- **Status Reporting** — Green/Yellow/Red status with context
- **Risk Communication** — ROAM framework (Resolved, Owned, Accepted, Mitigated)
- **Decision Documentation** — ADRs (Architecture Decision Records)
- **Meeting Facilitation** — Stand-ups, planning, retros, stakeholder reviews

## Output Format

Audience-tailored format. Scannable with bold headers and bullets. Executive: <300 words.

## ใช้เมื่อ

- Write status รายสัปดาห์/เดือนให้ leadership
- Announce launch
- Escalate risk หรือ blocker
- Translate progress into exec-brief, engineering-detail, customer-facing versions

## เชื่อมโยงกับ

- Project tracker (pull roadmap, milestone status)
- Chat (share updates with team)
- Meeting transcription (pull context from meetings)
- Knowledge base (find decision docs)
