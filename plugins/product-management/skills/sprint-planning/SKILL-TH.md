# Sprint Planning — วางแผน Sprint

**ประเภท**: User-invoked (slash command)

**Trigger**: `/sprint-planning [sprint name or date range]`

## ที่เก็บข้อมูล

วางแผน sprint — กำหนด scope, ประเมิน capacity, ตั้งเป้าหมาย และร่าง sprint plan

## คุณสมบัติหลัก

- **Capacity Estimation** — Available days, allocation, accounting for PTO/meetings
- **Sprint Backlog** — Items prioritized (P0/P1/P2), estimated effort, owners
- **Sprint Goal** — One clear sentence about success
- **Risk Identification** — Blockers, dependencies, mitigation plans
- **Definition of Done** — Code review, tests, documentation, sign-off
- **Carryover Management** — Handle unfinished items from last sprint

## Output Format

Structured template: Sprint plan with capacity table, backlog table, risks, definitions

## ใช้เมื่อ

- Kick off new sprint
- Size backlog against team availability
- Decide P0 vs stretch items
- Handle carryover from last sprint

## เชื่อมโยงกับ

- Project tracker (pull backlog, create sprint, assign items)
- Calendar (account for PTO and meetings)
- Chat (share sprint plan with team)
